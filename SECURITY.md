# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

- Do not disclose the issue publicly until it is fixed.
- Open a GitHub issue with the label `security`.
- Include a detailed description of the vulnerability and steps to reproduce.

## Supported Versions

This project is actively maintained for the current `main` branch.

## Security Practices

- Use trusted dependencies only.
- Avoid exposing sensitive credentials in code.
- Keep any deployed environment variables secure.
