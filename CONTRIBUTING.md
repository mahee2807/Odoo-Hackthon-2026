# Contributing to Odoo-Hackthon-2026

Thank you for your interest in contributing! This document explains how to contribute code, documentation, and ideas.

## How to contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature-name`.
3. Make your changes.
4. Commit with clear messages.
5. Push your branch and open a pull request.

## Code style

- Keep HTML/CSS/JS readable and consistent.
- Follow existing naming conventions for files and classes.
- Use descriptive commit messages.

## Issues

If you find a bug or want to request a feature:

- Open a new GitHub issue.
- Explain the problem clearly.
- Include reproduction steps and screenshots when possible.

## Pull requests

- Make sure your branch is up to date with `main`.
- Provide a description of what your change does.
- Link related issues or PRs.
- Keep your PR limited to a single purpose when possible.

## Testing

Test your changes locally before submitting.

- Run the frontend locally with `python -m http.server 8000` from `Ecosphere/frontend`.
- If you update backend code, run it with Node and verify the endpoints.

## Code of conduct

By contributing, you agree to follow the project Code of Conduct in `CODE_OF_CONDUCT.md`.
